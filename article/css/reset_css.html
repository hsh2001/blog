reset.css, CSS의 초기화.

CSS를 초기화한다?

웹 페이지는 한번 만들어두면 컴퓨터, 스마트폰, 태블릿 심지어 닌텐도까지도 웹 브라우저가 설치된 기기에서 접속하게 할 수 있다. 하지만 여기서 문제는 이토록 다양한 사용자가 하나의 브라우저로만 접속하는 것이 아니라는 것이다. 사파리, 크롬, 오페라, 파이어폭스 그리고 골치덩어리 인터넷 익스플로러 등등..

각각의 브라우저는 W3C 표준을 따르면서도 각자의 개성을 잘 살리기 위해 계속해서 개발되고 있다. 참고로 인터넷 익스플로러는 예외이다. 2013년 4분기에 마지막 버전인 11버전을 내놓고 개발사인 마이크로소프트는 2015년 신형 브라우저 엣지를 내놓으며 이젠 인터넷 익스플로러는 보안이나 간단한 버그 업데이트정도만 될 뿐이다. 하지만 한국에서의 사용률이 유난히 높기 때문에 울며 겨자먹기로 호환성 체크를 해줘야한다.

이러한 이유로 각각의 브라우저는 버전별로 종류별로 조금씩 다르게 동작하며 이 때문에 HTML, CSS, JavaScript가 예상과 다르게 작동하는 상황이 발생할 수 있다. 때문에 이에 대한 해결책으로 리셋(reset)과 폴리필(polyfill)을 사용한다.

폴리필은 다른 글에서 자세히 다루도록 하고, 리셋에 대해 알아보도록 하자. 리셋이란 말 그대로 다시 설정하는 것 이다.

이것도 리셋.

body { margin: 0; }

HTML body 요소에는 기본적으로 margin의 값이 8px로 주어진다. 만약 당신이 이 사실을 미리 알고 있어서 단순히 여백을 지우기 위해 이 코드를 사용한 적이 있거나 여백이 있는 상태가 좋아서 margin 값을 초기화하지 않은 경우, 이 글을 유심히 볼 필요가 있다.

보기 싫은(?) margin 값을 줄이기 위한 것이 목적이 되어서는 안된다는 말은 아니다. 공식 문서에는 body 요소에 margin: 8px, display: block을 초기값으로 갖는다고 하지만, 어느 브라우저는 이를 따르고 있지 않을 수도 있다.

예를 들면 표준을 따르지 않는 심술궂은 브라우저가 body 요소에 margin: 1rem 이라는 값을 주었다고 한다면, margin 값이 8px이었던 브라우저와 1rem이었던 브라우저와 다른 화면 구성을 갖게 될 것이며 이는 사용자에게 좋은 경험이 아닐 것 이다. 그림이 엉뚱한 위치에 있다던가 글이 화면을 뚫는 등...

하지만 이런 것은 body 요소에만 해당되는 이야기가 아니다. a, p, li, head... 때문에 많은 요소들의 css 속성을 다시 한번 적어줌으로써 초기화해주는 작업을 해줄 필요가 있고, 이 파일 또는 작업 따위를 <reset.css>라고 한다.

그렇다면 이러한 작업을 위해 그 많은 HTML 요소들을 전부 공부해야할까? 그럴 필요없다. 이미 열심히 공부한 다른 이가 reset.css를 만들어 배포하기 때문이다. 구글에 reset.css를 검색해서 적당한 css 파일을 다운받고 (되도록이면 최신 버전을 선택하자.) 이를 내 모든 웹 페이지에 적용시키면 된다. (저작권에 유의할 것!!!) reset.css가 적용됨에 따라 몇 요소의 css 속성이 변경되어 있을 수 있으니 전체적인 디자인 리워크도 필요할 것 이다.

이러한 작업을 통해, 내 웹 페이지가 다른 브라우저에서도 똑같은 화면구성을 만들어낼 것 이다. 웹 브라우저가 많아지고 발전하는 만큼 웹 개발자는 이에 따라 크로스 브라우징을 지원해야한다.
